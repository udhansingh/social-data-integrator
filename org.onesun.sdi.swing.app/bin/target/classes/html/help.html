Help

Social Data Integrator (SDI) is a tool to work with social-media providers such as Facebook, Twitter, LinkedIn and others.  The tool is useful for sending requests and receiving responses, additionally it allows one to model metadata based on request/response.  The metadata can further be used in other tools for processing social data.  Artifacts such as metadata, workflows, schedules generated by SDI can be published to a crawling service for further processing.  A Crawling service (will be available soon) may be used to periodically retrieve data from social media providers, process and perhaps store for further analysis.

Bootstrapping

Environment

Set environment variable SDI_HOME to a directory where you want to store configurations and other artifacts generated by SDI.

Example: SDI_HOME=C:/sdi/workspace

 

Services

SDI allows consumption of services for analysis.  You need to get your licenses for UClassify and OpenCalais.  This can be setup using “Settings”.

 

SDI uses embedded HTTP server to manage OAUTH handshake between you, SDI and provider.  Refer to your provider API and guidelines to obtain application keys and secrets.

 

Connectivity

Connecting to a provider is the first step to use the tool.  There are various connections that one can establish, depending on needs; a user will create appropriate connections.  In most cases, one must provide details to connection properties.  In case of Facebook, Twitter, LinkedIn and the likes, the tool uses OAUTH  protocol for authentication, hence the user must authorize the tool by creating necessary details in the provider systems before continuing to use SDI.
